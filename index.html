Autonomous Response

Three-Action Remediation Vocabulary

Different anomaly signatures trigger different actions. Confidence-gated. Cooldown-protected. Every action fully audited.

01
🔄
RESTART
Triggered by: error_burst, new_template, frequency_spike

Confidence gate: > 0.60
Cooldown: 60 seconds
docker.containers.get(svc).restart(timeout=10)
02
⬆️
SCALE UP
Triggered by: resource_saturation, throughput_anomaly

Confidence gate: > 0.75
Adds 1 replica via Compose
docker compose up --scale svc=N+1 -d
03
🛡️
CPU THROTTLE
Triggered by: latency_spike, multi_signal anomaly

Confidence gate: > 0.70
Limits runaway CPU usage
docker update --cpus 0.5 <container>
CONFIDENCE THRESHOLD SYSTEM
HIGH CONFIDENCE · IMMEDIATE ACTION ≥ 0.85
STANDARD CONFIDENCE · EXECUTE 0.60 – 0.85
LOW CONFIDENCE · BLOCKED < 0.60
